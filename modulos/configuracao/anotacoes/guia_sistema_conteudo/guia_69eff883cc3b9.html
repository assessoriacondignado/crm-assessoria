1. Objetivo
  • Permite que o usuário faça consulta manual do Consignado CLT do V8, obtendo: Concentimento, Aprovação Concentimento, Margem, Simualação padrão, Digitação, Acompanhamento.
2. Dono da chave
  • Cada Usuário do sistema possui uma chave de acesso que é preechida do Ambiente (Chave e Acesso)
  • Para que o usuário consulté é necessário ter saldo registrado em seu acesso
3. Cadastro cliente
  • Para Efeutar a cosnulta do Cliente na plataforma do V8, é ncessário ter os dados de: CPF, NOME, DATA NASCIMENTOE E SEXO
  • Para Clientes pré-cadastrado no banco de dados, o CRM localiza a informações e faz o pedido sem a necessidade do usuario preencher dados do cliente
  • Para Cliente aplicado Aletatoriamente sem pré-cadastro, o usuário pode:
    • Prencher os dados cadastrais manualmente (não necessita preecher e-mail ou telefone)
    • Utilizar a a ferramenta dados cadastrais

4. Fluxo consentimento / simulação padrão
  • Ao clicar em concentimento o CRM, envia o pedido para o sistema do V8 junto com o pedido de aprovação do consentimento
  • O V8 envia a solicitação para a Data Prev, onde aguardar seu retorno com o valor da Margem
  • O V8 responde ao CRM, que em seguinda efetua a consulta com a simualação padrão
  • Apresentando o usuário a simulação pronta com: praz, valor parcela e valor liquido
