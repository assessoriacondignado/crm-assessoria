1. Objetivo:
  • Aplciar parametos para o funcionamento da ferramenta
  • Usuário, Acesso Banco, paramentros de consulta
2. Username API / Password API
  • Deve ser aplicando o usuário e senha que seu usuário utiliza para consultar no site do banco
4. Custo p/ Cliente (R$) / Seu Custo API (R$)
  • Por padrão o sistema gerar modelo de cobrança para uso da ferramenta, no entendo esse processo é ajustado para não curto ao usuário e ser utilizado como controle interno
5. Tabela Padrão / Prazo Padrão / Averbadora: 
  • O usuário pode selecionar a tabela e o prazo padrão para simulação
  • O processo de cosnulta para CLT gera inicialmente uma simulação padrão, sendo enviado ao usuário no ato que é feito a consulta da margem
6.Intervalo entre Consentimentos (segundos):
  • A aplicação do tempo entre consentimentos é destinado a consulta em lote.
  • Para evitar acavalamento nas contas, o sistema possibilidade ajustar o tempo de consulta no lote
  • Por padrão é aplicando de 10 a 30 segundos
