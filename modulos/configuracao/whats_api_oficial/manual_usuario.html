Visão Geral do Módulo

Entendendo o WhatsApp Cloud API e como as credenciais se organizam

1.1 O que é o WhatsApp Cloud API?

O WhatsApp Cloud API (Oficial) é o canal de comunicação empresarial via WhatsApp fornecido pela Meta. Diferente de integrações não oficiais, ele:

  • É aprovado e hospedado pela própria Meta — não há risco de bloqueio de número.
  • Permite envio de templates aprovados a qualquer momento.
  • Permite envio de mensagens livres dentro de uma janela de 24h (após o cliente enviar qualquer mensagem).
  • Oferece webhook para receber mensagens e atualizações de status em tempo real.

1.2 Hierarquia de Credenciais

Antes de usar qualquer funcionalidade, é preciso cadastrar as credenciais na seguinte estrutura:

🏢 BM — Business Manager (Gerenciador de Negócios)
Contém: App ID, App Secret, Token Permanente, Token do Webhook.
Uma empresa pode ter mais de um BM.
💼 WABA — WhatsApp Business Account
Conta de negócios do WhatsApp vinculada ao BM. Um BM pode ter várias WABAs.
📱 Phone ID — Número de Telefone
ID do número ativo. Uma WABA pode ter vários números (ex: Atendimento, Vendas).
📌 Onde obter esses dados? Acesse business.facebook.com → Configurações → selecione a Conta do WhatsApp para ver o WABA ID e Phone Number ID. O Token Permanente é gerado em developers.facebook.com → Seu App → Ferramentas → Token de Acesso do Sistema.

Permissões de Acesso

Controle granular por aba e por hierarquia de dados

2.1 Chaves de Permissão Disponíveis

O sistema usa um modelo de bloqueio: quando uma chave é atribuída a um usuário, aquela aba ou função fica oculta. Sem nenhuma restrição, o usuário vê tudo.

SUBMENU_OP_WHATOFICIAL_CREDENCIAIS_DA_META
🔐 Credenciais da Meta
Oculta a aba de cadastro de BM, WABA e Phone IDs.
SUBMENU_OP_WHATOFICIAL_MODELOS
📋 Modelos
Oculta a visualização dos templates aprovados na Meta.
SUBMENU_OP_WHATOFICIAL_DISPARO_MANUAL
✉️ Disparo Manual
Oculta o formulário de envio avulso de mensagens.
SUBMENU_OP_WHATOFICIAL_CAMPANHA_LOTE
📣 Campanha / Lote
Oculta importação e disparo em massa.
SUBMENU_OP_WHATOFICIAL_HISTORICO
📜 Histórico
Oculta o log de mensagens enviadas e recebidas.
SUBMENU_OP_WHATOFICIAL_CHAT
💬 Chat
Oculta o painel de atendimento em tempo real.
SUBMENU_OP_WHATOFICIAL_MEU_CADASTRO
👤 Somente meu cadastro
Limita campanhas: usuário vê apenas os seus próprios registros.
SUBMENU_OP_WHATOFICIAL_CAMPANHA_HIERARQUIA
🏢 Hierarquia empresa
Limita campanhas ao escopo da empresa (não vê outras empresas).

2.2 Como Aplicar uma Permissão a um Usuário

1
Acesse: Configurações → Usuários → Permissões Localize o usuário na lista e clique em "Gerenciar Permissões".
2
Busque pela chave desejada Use o filtro para localizar SUBMENU_OP_WHATOFICIAL_ e marque as que devem ser bloqueadas para aquele usuário.
3
Salve e peça ao usuário relogar As abas bloqueadas desaparecem automaticamente ao recarregar o módulo.
⚠️ Atenção: Usuários do tipo Master/Admin têm acesso total e não são afetados pelas permissões de bloqueio.

Credenciais da Meta — Configuração do Webhook

Receber mensagens e status de entrega em tempo real

3.4 Configurar o Webhook na Meta

O webhook é o canal pelo qual a Meta envia ao CRM as mensagens recebidas dos clientes e as atualizações de status (entregue, lido, erro). Sem o webhook, o Chat e o Histórico não funcionam.

1
Copie a URL do Webhook Na aba Credenciais, copie a URL exibida no banner verde. Ela tem o formato:
https://seudominio.com.br/modulos/configuracao/whats_api_oficial/webhook_meta.php
2
Acesse: Meta for Developers → Seu App → WhatsApp → Configuração Role até a seção "Webhooks" e clique em "Editar".
3
Cole a URL e o Verify Token No campo Callback URL cole a URL do CRM. No campo Verify Token informe o Token de Verificação gerado no cadastro do BM. Clique em "Verificar e Salvar".
4
Assine os campos de webhook Após salvar, assine os campos: messages e message_status_updates.
developers.facebook.com → Seu App → WhatsApp → Configuração → Webhooks

✅ Tudo certo quando: A Meta exibe "Verificado" ao lado da URL. A partir daí, toda mensagem recebida de clientes aparece no Chat do CRM em segundos.
⚠️ O Webhook pertence ao BM/App, não à conta de WhatsApp individual. Uma única URL de webhook pode atender todos os Phone IDs do mesmo App.

4.2 Estrutura de um Template com Variáveis

Templates podem conter variáveis no formato {{1}}, {{2}}. Exemplo:

Olá, {{1}}! Temos uma proposta especial para você:
Margem disponível: R$ {{2}}
Parcela: R$ {{3}} — {{4}} meses.
Responda SIM para saber mais!

Na aba Disparo Manual e na Campanha/Lote, essas variáveis são preenchidas automaticamente com os dados de cada cliente (nome, margem, parcela, prazo).

5.1 Enviar Mensagem de Texto Livre

1
Selecione o remetente Escolha qual número oficial da empresa irá enviar.
2
Preencha o número destino Sempre com DDI + DDD + número. Ex: 5562999998888 para Goiás-Brasil.
3
Selecione "Texto Livre (Janela 24h)" Digite o conteúdo da mensagem. Atenção: só funciona se o cliente tiver enviado alguma mensagem nas últimas 24h.
4
Clique em "Enviar" O sistema exibe o resultado (✅ Enviado ou ❌ Falha) abaixo do botão.

5.2 Enviar via Template Aprovado

1
Selecione "Template Aprovado" no campo Tipo O formulário adapta campos extras: seletor de template, URL de imagem (opcional) e variáveis.
2
Escolha o template na lista Apenas templates com status APPROVED aparecem disponíveis.
3
Preencha as variáveis separadas por vírgula Se o template tem {{1}}, {{2}}, {{3}}, informe os valores na mesma ordem. Ex: João, 1500.00, 250.00
⚠️ Janela 24h: Envios de texto livre falham se o cliente não tiver interagido nas últimas 24h. Nesse caso, use sempre um Template Aprovado para retomar a conversa.

6.1 Importar Lista pela Tela (Cola de Números)

1
Preencha Nome, Remetente e Template Escolha o template aprovado que será enviado para todos da lista.
2
Selecione "Colar na Tela" e cole os números Um número por linha, com DDI e DDD. Ex: 5562999998888
3
Clique em "Criar Campanha" Os números são importados e ficam com status PENDENTE. A campanha aparece na tabela à direita.

6.2 Importar Lista via Arquivo CSV

Para envios com dados personalizados (nome, margem, parcela, prazo), use o CSV com o seguinte formato:

NUMERO; NOME; CPF; MARGEM; PARCELA; PRAZO; VALOR_LIQUIDO
5562999998888; João Silva; 12345678900; 1500.00; 250.00; 6; 1480.00
5511988887777; Maria Souza; 98765432100; 2000.00; 310.00; 8; 1950.00
📁 Formato do CSV: Separador ponto-e-vírgula (;), sem aspas. A primeira linha é o cabeçalho e é ignorada na importação. O CPF é mascarado automaticamente no sistema.

6.3 Processar e Acompanhar o Disparo

1
Clique em "▶ Processar" na campanha desejada O sistema processa em lotes de 10 mensagens por vez. O progresso atualiza automaticamente.
2
Acompanhe os contadores: Enviados / Falhas / Total A barra de progresso avança a cada lote concluído.
3
Ao concluir, o botão mostra "✅ Finalizado" Clique em "📥 Exportar" para baixar um CSV com o resultado completo (status por número).

Aba: Chat

Atendimento em tempo real com clientes via WhatsApp

O Chat é o painel de atendimento ao vivo. Ele exibe as conversas organizadas por status, mostra os balões de mensagem e permite ao atendente responder diretamente pelo CRM.

8.1 Regra da Janela de 24 Horas (Meta)

A Meta impõe uma regra fundamental para o uso da API:

🟢 Janela ABERTA

Condição: O cliente enviou alguma mensagem nas últimas 24 horas.

O que posso enviar:

  • ✅ Texto livre (qualquer conteúdo)
  • ✅ Imagens, áudios, documentos
  • ✅ Templates aprovados

A janela se renova a cada mensagem recebida do cliente.

🔴 Janela FECHADA

Condição: O cliente não enviou mensagem há mais de 24 horas.

O que posso enviar:

  • ❌ Texto livre — BLOQUEADO pela Meta
  • ❌ Imagens/arquivos avulsos
  • ✅ Apenas Templates aprovados

Envie um template para reabrir a janela.

✅ Comportamento no CRM: O sistema exibe um banner colorido indicando o status da janela. Quando fechada, o campo de texto é desativado automaticamente — apenas o seletor de templates permanece disponível.

Chat — Funções de Atendimento

Iniciar conversa, enviar mensagens e gerenciar atendimentos

8.3 Iniciar uma Nova Conversa

Para entrar em contato com um cliente que ainda não iniciou conversa:

1
Clique no botão "+" (verde) no topo da lista O modal "Nova Conversa" será exibido.
2
Selecione o número remetente e informe o telefone do cliente O número deve incluir DDI e DDD. Ex: 5562999998888
3
Clique em "Iniciar Conversa" Uma conversa é criada na lista. Como a janela estará fechada (o cliente nunca escreveu), será necessário enviar um template aprovado como primeira mensagem.

8.4 Enviar Mensagem de Texto (Janela Aberta)

1
Selecione a conversa na lista Verifique que o banner indica 🟢 Janela aberta.
2
No seletor de tipo, escolha "texto" O campo de digitação estará ativo.
3
Digite e envie com Enter ou botão ➤ O balão aparece imediatamente do lado direito (enviada). O status atualiza conforme a entrega.

8.5 Enviar Template (Janela Fechada ou Iniciar Contato)

1
No seletor de tipo, escolha "template" Um seletor adicional aparece com os templates aprovados disponíveis.
2
Selecione o template desejado e clique em ➤ Enviar O template é enviado imediatamente. Se o cliente responder, a janela de 24h abre e você poderá enviar texto livre.

8.6 Enviar Arquivo / Mídia

1
No seletor, escolha "arquivo" Um botão de seleção de arquivo é exibido no lugar do campo de texto.
2
Selecione o arquivo e clique em ➤ Tipos aceitos: imagem (jpg, png, gif, webp), vídeo (mp4), áudio (mp3, ogg, opus) e documento (pdf). Limite: 16MB por arquivo.
⚠️ Atenção: Envio de arquivos só é possível com a janela aberta. Fora da janela, use apenas templates.

8.7 Modelos CRM (Respostas Rápidas)

Modelos CRM são textos salvos dentro do próprio CRM (diferentes dos templates da Meta). Servem para agilizar respostas padrão frequentes.

1
Clique em "📁 Modelos CRM" na área de input O modal de modelos rápidos abre com a lista de textos cadastrados.
2
Clique em "Usar" ao lado do modelo desejado O texto é copiado para o campo de mensagem. Você pode editar antes de enviar.
3
Para criar um novo modelo: clique em "+ Novo Modelo" Defina um nome e o conteúdo. Os modelos ficam disponíveis para todos os atendentes da empresa.

8.8 Finalizar Atendimento

1
Clique em "Finalizar" no cabeçalho da conversa O status da conversa muda para FINALIZADO e ela sai do filtro padrão.
2
Se o cliente responder após finalizar A conversa reabre automaticamente com status AGUARDANDO e aparece na lista novamente.

Glossário

Termos técnicos utilizados no módulo

TermoDefinição
Meta Empresa controladora do Facebook, Instagram e WhatsApp. Fornece a API oficial para envio de mensagens.
WhatsApp Cloud API API oficial da Meta hospedada na nuvem deles. Diferente da API On-Premise, não requer servidor dedicado.
BM (Business Manager) Gerenciador de Negócios da Meta. Agrupa o App, credenciais e contas de WhatsApp de uma empresa.
App ID / App Secret Identificadores do aplicativo criado no Meta for Developers. O App é necessário para autenticar chamadas da API.
Token Permanente Chave de autenticação de longa duração gerada para um usuário do sistema no Meta for Developers. Necessário para todas as chamadas.
WABA WhatsApp Business Account. Conta de negócios que agrupa os números de telefone oficiais.
Phone Number ID Identificador numérico do número de telefone cadastrado na WABA. Diferente do número de telefone em si.
Webhook URL do CRM que a Meta chama automaticamente quando há uma mensagem nova ou atualização de status.
Verify Token Token cadastrado no BM e informado à Meta para autenticar o webhook no momento da configuração.
Template Mensagem pré-aprovada pela Meta que pode ser enviada a qualquer momento, inclusive fora da janela 24h.
Janela 24h Período de 24 horas após a última mensagem do cliente no qual é permitido enviar texto livre. Após esse prazo, apenas templates.
DDI Código internacional do país. Brasil: 55. Sempre incluir antes do DDD e número ao enviar mensagens.
Modelo CRM Texto de resposta rápida salvo no CRM (diferente dos templates da Meta). Não precisam de aprovação.
VerificaBloqueio Função interna do CRM. Retorna verdadeiro quando o usuário tem aquela chave de permissão bloqueada.
